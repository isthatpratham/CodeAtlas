# 1. Purpose

This document defines every major screen and UI component used throughout CodeAtlas.

The goal is to ensure every interface follows a consistent structure, interaction model, and visual hierarchy.

This document specifies **what the user sees and interacts with**, while implementation details are covered elsewhere.

---

# 2. UI Philosophy

The interface should prioritize the repository map over everything else.

The map is the product.

Every surrounding panel exists only to support exploration, never compete with it.

Design principles:

* Minimal
* Spacious
* Calm
* Fast
* Professional
* Content-first
* Non-distracting

Users should always know where to focus.

---

# 3. Application Screens

Version 1 consists of only two primary screens.

## Landing Page

Purpose:

Import a repository.

---

## Workspace

Purpose:

Explore the repository.

No additional pages should interrupt this workflow.

# 4. Landing Page

The landing page should be simple and focused.

Layout:

```text
---------------------------------------------------

                 CodeAtlas Logo

      Visualize any GitHub repository
        like an interactive map.

   ┌──────────────────────────────┐
   │ Paste GitHub Repository URL  │
   └──────────────────────────────┘

         [ Explore Repository ]

-----------------------------------------------

Example Repositories

-----------------------------------------------

Features Preview

-----------------------------------------------

Footer
```

### Primary CTA

**Explore Repository**

No unnecessary options should distract from the main action.

---

### Example Repositories

Provide several popular repositories users can instantly explore.

Examples:

* React
* Next.js
* Vue
* Express
* Tailwind CSS

Selecting one should automatically populate the URL field.

---

### Features Preview

A concise overview of:

* Interactive Map
* Dependency Graph
* Search
* Repository Statistics
* Smooth Navigation

---

# 5. Workspace Layout

The workspace is the core experience.

```text
─────────────────────────────────────────────

Top Navigation

─────────────────────────────────────────────

Sidebar      Interactive Map      Inspector

              Mini Map

─────────────────────────────────────────────

Status Bar

─────────────────────────────────────────────
```

The interactive canvas should occupy approximately 70–80% of the viewport.

# 8. Interactive Canvas

The canvas is the centerpiece of CodeAtlas.

Users can:

* Zoom
* Pan
* Select
* Hover
* Explore

The canvas should always remain smooth.

---

### Folder Nodes

Display:

* Folder Name
* File Count

---

### File Nodes

Display:

* File Name
* Extension

---

### Hover

Hovering reveals:

* File Path
* Language
* Dependency Count

---

### Selection

Selecting a node:

* Highlights related nodes
* Opens Inspector
* Centers camera if required

---

### Background

Subtle dotted grid.

No decorative illustrations.

---

## Relationships

Child Files

# 10. Search System

Search should be globally accessible.

Users should be able to search:

* Files
* Folders

Search should be:

* Instant
* Keyboard-friendly
* Fuzzy matching capable

Results should:

* Highlight matching nodes
* Move camera
* Open Inspector

# 15. Keyboard Shortcuts

Version 1 includes:

| Shortcut     | Action              |
| ------------ | ------------------- |
| /            | Focus Search        |
| Esc          | Close Inspector     |
| F            | Focus Selected Node |
| +            | Zoom In             |
| -            | Zoom Out            |
| 0            | Reset Zoom          |
| Space + Drag | Pan Canvas          |

Future shortcuts can be added without changing the interaction model.
